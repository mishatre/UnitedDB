Обработка «Импорт/Экспорт контактных данных объектов»

Обработка предназначена для обмена контактной информацией и данными объектов с программами Outlook Express, Microsoft Outlook и The BAT! посредством текстовых файлов в формате CSV (значения разделенные запятыми) или TDF (значения разделенные табуляциями).

В качестве объектов 1С:Предприятия могут выступать элементы справочников:

  • Контрагенты
  • Контактные лица контрагентов
  • Прочие контактные лица
  • Личные контакты
  • Физические лица

После настройки всех значений на основной форме, таких как формат файла, путь к файлу, программа импорта/экспорта, вид операции и тип объекта 1С:Предприятия, по нажатию на кнопку «Выполнить» нижней командной панели формы, откроется окно дальнейшей настройки обмена.

Импорт (загрузка)

1-й шаг. Настройка соответствия полей файла и контактных данных объектов.

Формируется таблица, где в колонке «Имя поля из файла» отображаются имена полей данных для загрузки в базу. Пользователь флажками должен указать поля, которые необходимо загружать. При этом в поле таблицы «Имя или вид реквизита в информационной базе» необходимо указать либо реквизит соответствующего справочника, который является объектом для загрузки, либо выбрать вид контактной информации для загрузки данных.

Если установить флажок «Сохранять соответствие параметров и настройки поиска объектов для последующих загрузок» то при последующих загрузках настройки сделанные на всех шагах процесса настройки загрузки данных, будут восстановлены.

Правила формирования наименования загружаемых объектов назначаются в отдельной форме, которая открывается при нажатии на кнопку «Настройка формирования наименования объекта».

2-й шаг. Настройка критериев поиска объектов в текущей информационной базе.

Пользователь может настроить критерии, по которым перед загрузкой, будет произведен поиск уже существующих объектов в информационной базе 1С:Предприятия. Состав полей, по которым возможен поиск и идентификация объектов, пользователем не меняется. Настраивается только условие (равно или не равно) и выбирается поле из файла.

3-й шаг. Настройка загрузки данных в текущую информационную базу.

При переходе к данному шагу, производится поиск объектов по критериям настроенным на 2-м шаге. Заполняется таблица. В поле «Наименование объекта в ИБ» пользователь может откорректировать наименование, с которым будут создаваться новые объекты в информационной базе. В поле «Объект в ИБ» программа сама проставит найденный объекты, можно выбрать вручную. Для строк с на выбранными объектами, они будут создаваться. Флажками отмечаются те объекты, данные по которым необходимо загружать.

Если производится загрузка контактных лиц контрагентов, то появляется дополнительное поле «Контрагент», в котором обязательно указывать контрагента-владельца контактного лица, данные по которому будут загружаться.

Стоя на любой строке таблицы при нажатии на кнопку «Посмотреть информацию из файла по текущей строке» командной панели таблицы, можно просмотреть информацию которая будет загружаться по выбранному объекту.

Можно указать группу, которая проставится в качестве родителя для создаваемых элементов.

При нажатии на кнопку «Готово» нижней командной панели формы производится загрузка данных.

Экспорт (выгрузка)

1-й шаг. Настройка списка полей выгрузки.

Отображается таблица, в которой указаны данные, возможные для выгрузки из информационной базы 1С:Предприятия для выбранного объекта обмена. Флажками отмечаются те, которые необходимо выгружать.

2-й шаг. Настройка списка объектов выгрузки.

В таблице пользователю необходимо указать, данные каких именно объектов необходимо выгружать. Если выбран тип объекта выгрузки «Контактные лица контрагентов» то в таблице дополнительно появляется колонка «Контрагент».
Можно воспользоваться кнопками командной панели табличного поля для множественного подбора или для автоматического заполнения таблицы всеми существующими объектами в информационной базе. По кнопке «Очистить» все строки табличного поля удаляются.
При нажатии на кнопку «Готово» нижней командной панели формы производится выгрузка данных.
