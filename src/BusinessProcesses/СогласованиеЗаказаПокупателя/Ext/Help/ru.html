Бизнес-процесс "Согласование заказа покупателя"

Бизнес-процесс «Согласование заказа покупателя» служит для формализации операций по согласованию заказов покупателей сотрудниками предприятия.

Процесс согласования состоит из следующих этапов:

  • согласование заказа
  • подведение итогов согласования
  • ознакомление с итогами согласования
  • начать выполнение заказа

Этапы согласования отражены на карте маршрута бизнес-процесса в виде точек «действия». Бизнес-процесс, следуя по маршруту, автоматически в каждой точке «действия» создает задачи. 

На этапе согласования заказа исполнителям предлагается ввести визы и рецензии на согласуемый объект. После выполнения задач в этой точке маршрута формируется задача на подведение итогов согласования. Исполнитель этой точки действия на основе информации о полученных визах принимает итоговое решение. Если результат согласования положительный - создается задача пользователю с извещением об удачном завершении согласования, при выполнении которой бизнес-процесс завершается. Если результат согласования отрицательный - то создается задача, в которой пользователь может повторить согласование или завершить процесс.

Ввести новый экземпляр бизнес-процесса можно путем ввода на основании документа «Заказ покупателя». Вариант прохождения бизнес-процесса по маршруту выбирается в поле "Настройка"  из справочника "Настройки бизнес-процессов" . В настройке для каждой точки «действия» необходимо указать исполнителя. В качестве исполнителя может быть указан как пользователь, так и инициатор бизнес-процесса. Остальные параметры - представление, детальное описание, планируемый период выполнения и оповещения задачи не являются обязательными.  При необходимости некоторые точки можно отключить, что приведет к сокращению стандартной карты маршрута бизнес-процесса.

Для получения информации о текущей точке маршрута бизнес-процесса можно использовать «Карту маршрута», которая вызывается по нажатию кнопки  в командной панели формы.

С помощью отчета «Анализ согласования заказов покупателей»  можно получить информацию о текущем состоянии согласования, визах и рецензиях пользователей. Отчет вызывается по кнопке «Анализ» командной панели формы.

По одному заказу покупателя может быть запущено неограниченное количество бизнес-процессов согласования.
