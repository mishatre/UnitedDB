Документ «План закупок»

Документ предназначен для отражения в управленческом учете следующих операций:

  • Ввод в систему и хранение данных по предполагаемым объемам закупок
  • Формирование планов закупок по данным подсистемы планирования и фактическим данным смежных подсистем

Ввод в систему и хранение данных по предполагаемым объемам закупок

В документе может указываться оценка предполагаемых объемов закупок или для элементов номенклатуры (уточненное планирование) или для номенклатурных групп (укрупненное планирование). Оценка закупок указывается для отдельного периода по отдельному сценарию, при этом может производиться уточнение до подразделения и / или проекта.

Реквизиты «Сценарий», «Дата планирования» являются обязательными. Заполнение реквизита «Сценарий» определяет:

  • периодичность планирования, используемую в данном документе;
  • объект планирования – элементы номенклатуры или номенклатурные группы (ввод данных по одному сценарию планирования одновременно и по номенклатуре и по номенклатурным группам невозможен).

Реквизит «Дата планирования» выравнивается по периодичности планирования (на начальную границу).
Реквизиты «Проект» и «Подразделение» позволяют дополнительно отнести вводимые данные к какому-либо проекту или подразделению.

Состав плановых данных вводится в табличную часть документа. Объекты планирования (номенклатура / номенклатурная группы), указываются с точностью до характеристики, если соответствующие параметры номенклатуры ведутся. По каждому объекту указывается единица измерения, плановый объем закупок в количественной (если по сценарию ведется планирование количества закупок) оценке и суммовой (если по сценарию ведется планирование сумм закупок) оценке с выделением НДС закупок.

Валюта документа, тип цен, признаки учета НДС настраиваются штатным для документов образом – в диалоговом окне «Цены и валюта» документа.

Документа может заполняться штатным подбором из справочника «Номенклатура».

Формирование планов закупок по данным подсистемы планирования и фактическим данным смежных подсистем


Состав документа может быть заполнен в автоматическом режиме, для этого используется форма заполнения, вызываемая по кнопке «Заполнить».

В форме заполнения производится настройка правил заполнения документов.

Формирования количественных показателей настраивается на первой закладке. Возможные варианты заполнение документов: по фактическим закупкам за период, по плановым данным закупок и по заказам поставщикам. При заполнении может производиться учет текущих остатков на складах. Формируемый результат может быть скорректирован на произвольный коэффициент.

Формирование суммовых показателей настраивается на второй закладке. Суммы (для рассчитанных количеств) могут исчисляться по указанному типу ценам номенклатуры, по указанному типу цен контрагентов, по объему закупок за период в стоимостном выражении, или по стратегии, аналогичной заполнению количеств, как это настроено на первой закладке. Формируемые суммы также можно скорректировать на произвольный коэффициент.

Для фильтрации исходных для заполнения данных используется третья закладка. В ней можно накладывать условия на отбираемые для заполнения исходные данные – заказы, сценарии и т.п.

Собственно заполнение документа производится кнопкой «Выполнить».

Для записанного документа возможно формирование печатной формы.

Документ не используется для ведения регламентированного учета.
