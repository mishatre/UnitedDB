Настройка параметров учета

Настройка параметров учета вызывается из меню «Сервис» выбором пункта «Настройка параметров учета». В диалоговом окне присутствуют закладки, на которых задаются параметры, по которым будет вестись учет в компании.

Закладка «Общие»

На этой закладке устанавливаются флаги использования той или иной подсистемы в учете: использовать возвратную тару, использовать характеристики номенклатуры, использовать оплату платежными картами и т.д. При снятии того или иного флага в программе исключается возможность работы с подсистемой.

Пример. Если в настройках параметров учета отключить флаг "Использовать возвратную тару", то в документах не будет видна закладка "Тара". Пользователь не сможет ввести в документы информацию о возвратной таре и оформить новые документы "Изменение лимитов возвратной тары" и "Корректировка долга по возвратной таре". В то же время, если в документе уже была заполнена информация о таре, то эта закладка останется видимой в этом документе даже после отключения флага "Использовать возвратную тару".

Флаг "Использовать ограничение прав доступа на уровне записей" устанавливается в том случае, если необходимо установить права доступа пользователей к конкретным объектам (контрагентам, организациям). 

  • Интервал проверки напоминаний в секундах. Интервал, с которым будет проверяться наличие записей для напоминания пользователям при исполнении поставленных перед ними задач.
  • Способ контроля дней задолженности. В реквизите выбирается способ расчета дней задолженности, который будет использоваться при оперативном контроле взаиморасчетов с контрагентом. Срок задолженности может рассчитываться по календарным дням или по рабочим дням. Рабочие дни считаются в соответствии с производственным календарем, который действует на торговом предприятии. 
  • Указание складов в табличной части документов. Значение реквизита определяет возможность указания складов в табличной части документов поступления и реализации.
  •  Указание заказов в табличной части документов. Значение реквизита определяет возможность указания заказов покупателей  в табличной части документов поступления и реализации.
  • Списывать партии расходным ордером. Если флажок установлен, то документ "Расходный ордер на товары" будет производить списание партий товаров.
  • Настройка шаблонов «быстрой» регистрации нового контрагента. Пользователь может заранее задать поля для быстрой регистрации нового контрагента. Форма быстрой регистрации контрагента используется при вводе нового контрагента с помощью помощника ввода контрагентов.

Закладка «Заказы»

  • Стратегия авторезервирования по заказам. В реквизите выбирается стратегия авторезервирования по  заказам покупателей, которая будет использоваться в документах по умолчанию: «Сначала на складах, потом в заказах поставщикам» или «Сначала в заказах поставщикам, потом на складах».
  • Использовать указание серий при резервировании. Если флажок установлен, то резервирование товаров на складах по заказам покупателей можно выполнять с учетом серий товаров.

Закладка «Печать, единицы»

Дополнительная колонка печатных форм документов - информация в данном поле определяет ту дополнительную информацию, которая выводится в печатных формах документов и в отчетах.

  • Код - в печатных формах документов и в отчетах будет дополнительно выводиться код, заданный для позиции номенклатуры. 
  • Артикул - в печатных формах документов и в отчетах будет дополнительно выводиться артикул, заданный для позиции номенклатуры. 
  • Не выводить - дополнительной информации выводиться не будет.

Единица измерения веса -  в данном поле необходимо задать краткое наименование единицы веса, используемой для измерения веса позиции номенклатуры в конфигурации.

Единица измерения объема - в данном поле необходимо задать краткое наименование единицы объема, используемой для измерения объема позиции номенклатуры в конфигурации.

Тип цен плановой себестоимости. По указанному типу цен будет рассчитываться плановая себестоимость и отклонение от плановой себестоимости в документах "Заказ покупателя", "Заказ поставщику".

Закладка «Валюты»

Валюта регламентированного учета — базовая валюта, курс которой всегда равен 1 (для России —

рубли). Курсы всех других валют выражаются в единицах валюты бухгалтерского учета.

Валюта управленческого учета — валюта, используемая для оценки данных в управленческом учете. Обычно в качестве валюты управленческого учета указывают доллар США.

Закладка «Электронная почта»

Пользователь может указать вариант использования электронной почты. Можно пользоваться либо встроенным в конфигурацию почтовым клиентом, либо основным почтовым клиентом операционной системы (Microsoft Outlook, Outlook Express, The BAT! и пр.). Для встроенного почтового клиента задается администратор учетных записей, который может редактировать учетные записи других пользователей.

Закладка «Обмен данными»

На этой закладке задаются параметры для автоматического обмена данными при работе с распределенными информационными базами.

Префикс кодов справочников и документов задается, если предполагается обмен данными между распределенными информационными базами (РИБ). Префиксация позволяет избежать дублирование кодов элементов в узлах РИБ.

Префиксация осуществляется только если в списке узлов плана обмена "Полный" создан хотя бы один узел. В этом случае префикс добавляется к кодам при создании новых элементов справочников, документов и элементов планов видов характеристик. Установка или изменение префикса не приводит к перенумерации кодов уже существующих элементов.

Важно. Не рекомендуется в префиксах использовать цифры. Это может привести к ошибочной нумерации объектов или к ошибкам при обмене данными.

Префикс внутреннего штрихкода имеет то же назначение, что и префикс кодов справочников и документов. См. также: Общая информация о штрих-кодах.

Пользователь для регламентных заданий в файловом режиме определяет пользователя, в клиентском сеансе которого должны выполняться фоновые и регламентные задания в файл-серверном режиме работы информационной базы. Рекомендуется запускать отдельный клиентский сеанс от имени указанного пользователя. В частности, это позволяет производить автоматический обмен по расписанию. При этом пользователь должен обладать достаточными правами для выполнения произвольных операций по чтению и записи данных информационной базы.

Для клиент-серверного режима работы этот параметр задавать не требуется.

Интервал опроса регламентных заданий в файловом режиме также необходимо задавать только для файл-серверного режима работы информационной базы.

Коэффициент пересчета веса товаров в граммы используется для пересчета веса товаров в граммы для выгрузки данных на сайт (если коэффициент не задан, то считается равным 1).

Флажок Сохранять информацию о месте создания объектов информационных баз необходимо устанавливать для корректного двухстороннего обмена данными с другими конфигурациями (например: БП, РТ или УПП). При этом включаются ограничения на миграцию элементов справочников и документов: разрешается редактировать данные только в той информационной базе, в которой эти данные были созданы. Эти ограничения обеспечивают целостность данных в обоих обменивающихся информационных базах.

Услуга для загрузки неторговых операций используется для заполнения реквизита "Номенклатура" у документов, мигрирующих при обмене данными из конфигурации БП. См. также: "Поступление товаров и услуг", "Реализация товаров и услуг".

Закладка «Шаблоны телефонов»

Для единообразия вводимых телефонных номеров всеми пользователями в контакную информацию, можно задать единые шаблоны. При изменении или добавлении телефонных номеров, они будут приводиться к указанным шаблонам.

Закладка «Коды товара»

На этой закладке задаются параметры для автоматического формирования штрихкодов на товары.
В конфигурации предусмотрено автоматическое шрихкодирование товаров для типа штрихкода EAN-13. В качестве параметров задается максимальный текущий код для весового и штучного товара. В соответствии с этим кодом будет формироваться новый штрихкод для товара.

В том случае, если на торговом предприятии ведется учет по штрихкодам весового товара, то необходимо задать префикс штрихкода весового товара. По этому параметру при заполнении информации о весовом товаре с помощью сканера штрихкодов в документе будет определяться, что товар является весовым, и в документ будет заполняться вес товара, заданный в штрихкоде.

Заполнение префикса для штучного товара позволит формировать этикетки на товары с внутренним штрихкодом, отличным от штрихкода поставщика.
Для каждой единицы измерения товаров задается свой штрихкод. Если на предприятии допускается назначать разным товарам или товарам с разными единицами измерения один и тот же штрихкод, то необходимо установить флаг "Разрешить назначение неуникальных штрихкодов".
